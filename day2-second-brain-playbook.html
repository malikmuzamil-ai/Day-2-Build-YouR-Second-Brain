Two things, before anything else

Day 2 needs exactly two accounts. Get these sorted first and the rest is clicking. If you already have both from yesterday, jump straight to Deploy the Repo.

Account 1

A GitHub account

This is where our app gets copied so you own your own private copy. It is free, and you only need the login.

  1. Go to github.com/signup
  2. Enter your email, pick a password, and choose a username
  3. Open the verification email from GitHub and click the link inside
  4. Already have an account? Just log in at github.com/login instead
DONE →You can see your GitHub dashboard when logged in. That is all you need here.
Account 2

An Anthropic API key

This is the fuel. It is what lets your live brain think and write. You paste it once during deploy and never touch it again.

  1. Go to console.anthropic.com and sign in
  2. Click API Keys in the left menu
  3. Click Create Key and name it second-brain
  4. Copy the key the moment it appears on screen
  5. Paste it into your Notes app for a few minutes. You will need it during deploy
GUARD ITYour API key works like a credit card. Keep it private, and only paste it into the one deploy box shown in Stage 2. If anyone else gets it, they can spend on your account.

Copy the repo, then deploy it

This is the heart of Day 2. You will copy our app into your own GitHub, then hand it to the host with one click. The only thing you type is your API key. Each step below matches one screenshot, so follow them in order.

Stage 2 · Step 1
Step 1 · Open the repo

Go to the shared repository

This is the app we built for you. Everything the live brain needs is already inside it, so you just need your own copy.

  1. Make sure you are logged into GitHub first
  2. Open github.com/demo55oo/ai-second-brain
  3. You will see folders like content and src. Leave them as they are
  4. Scroll down to the README, which is where the deploy buttons live
Stage 2 · Step 2
Step 2 · Hit deploy

Click the deploy button

Scroll through the README until the deploy buttons appear. Clicking one copies the repo into your account and sets up the live site in a single move.

  1. Find the Deploy buttons inside the README
  2. Click one to begin
  3. When it asks, choose to connect your GitHub
  4. Read the note on screen: BRAIN_ENGINE=api is already set for you, so the only thing you will paste later is your API key
GOOD →Only one field matters: your API key. You can leave Supabase blank for today.
Stage 2 · Step 3
Step 3 · Connect GitHub

Create your private copy

The host asks where to put your copy of the app. Pick your own GitHub account so this becomes your private repo that only you control.

  1. Click Connect GitHub when prompted
  2. Approve the access request on the GitHub screen
  3. Under Git Scope, choose your own account
  4. Leave the repository name as ai-second-brain
  5. Keep the repository set to Private
  6. Click Create to clone it into your account
Stage 2 · Step 4
Step 4 · Paste your key

Add one environment variable

This is the only thing you type in the whole deploy. Paste your Anthropic API key into the value box. That is the fuel that makes your brain think.

  1. Find the Add Environment Variables screen
  2. Look at the Name field, which shows AI_GATEWAY_API_KEY
  3. Click the Value box and paste your Anthropic API key from Stage 1
  4. Paste the key on its own, with no label and no quote marks around it
  5. Leave BRAIN_ENGINE as it is, since it already defaults to api
COMMON SLIPIf the key is missing its first character or has an extra space, the site loads but chat fails with an API-key error. Re-paste the key carefully and that usually fixes it.
Stage 2 · Step 5
Step 5 · Deploy

Press Deploy and wait

Now the host builds your site. This takes about three minutes. You will see build logs scroll past, and a yellow warning or two along the way is normal.

  1. Click the large Deploy button
  2. Watch the Deployment panel show "Deployment started" and then the build logs
  3. Ignore any npm warn deprecated lines, since warnings are safe
  4. Wait about three minutes for the build to finish
Stage 2 · Step 6
Step 6 · You are live

Open your brain

When the deploy finishes you get a congratulations screen and a live link. Click it and your Second Brain opens in the browser, running on the internet on your own account.

  1. On the success screen, click your project link to open the live site
  2. Skip the "Install Coding Agent Plugin" step, since it is optional
  3. Bookmark the link, because this is your brain's home on the web
MILESTONEYour Second Brain is now live on the internet. The next stage swaps in your own knowledge so it writes as you.

Drop your .zip in

When something snags

Almost every snag in Day 2 comes down to the key, the account, the host, or the zip. Find your problem below, apply the fix, and keep moving. If none of these match, the rescue prompt at the bottom handles the rest.

Deploy problems

The site is live but chat fails

Error: invalid API key, or chat returns nothing

Cause: the key was pasted with a stray character, or its first letter got clipped. Fix: open your project's Settings, then Environment Variables. Edit AI_GATEWAY_API_KEY or ANTHROPIC_API_KEY, paste the value again cleanly, save, and click Redeploy.

Build failed with a red error

Cause: usually a half-finished GitHub connection. Fix: delete the project on the host and run the deploy button again from the repo README. A fresh clone clears it.

A yellow npm warn deprecated line

This is a warning, so it is safe to ignore. Only a red error will stop a deploy.

Account problems

GitHub will not connect

Cannot copy the repo, or the authorize screen keeps looping

Cause: your email is not verified, or you got signed out mid-flow. Fix: verify your GitHub email, log back in, and restart the deploy button. If you just restarted your laptop, sign in again first.

No Anthropic key yet

Fix: go to console.anthropic.com, open API Keys, and click Create Key. Copy it the instant it shows, because you cannot view it again later.

Upload problems

Counts stay at zero after upload

Notes, Chunks, and Folders still show 0

Cause: cloud uploads need a writable host or the optional Supabase keys. Fix for today: your chat still works, so you can test the flow now. To make the upload stick, add the optional Supabase env keys after deploy, then upload the zip again.

Upload rejected

Cause: wrong file type. Fix: the box accepts .md files or a single .zip vault. Check that you zipped the whole folder and that the files inside are markdown.
